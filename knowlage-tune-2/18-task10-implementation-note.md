# 18-task10 Implementation Note: Summary field semantics cleanup

## Problem

`EvidenceRef.summary_ru` was being populated from `compact_visual_summary`, which contains English-language visual descriptions generated by the dense analyzer. This meant `summary_ru` contained English text, violating the language guarantee implied by its name.

## Solution

Added language-aware summary fields and a simple detection heuristic so each field carries only the correct language content.

## New field semantics

| Field | Purpose |
|-------|---------|
| `compact_visual_summary` | Legacy/raw compact summary (unchanged, backward-compatible) |
| `summary_primary` | Canonical main summary, language-agnostic |
| `summary_language` | Detected language code: `"ru"`, `"en"`, or `None` |
| `summary_ru` | Only populated when content is actually Russian |
| `summary_en` | Only populated when content is actually English |

## Language detection

`detect_summary_language(text)` in `evidence_linker.py` uses a simple Cyrillic-vs-Latin character count:
- More Cyrillic -> `"ru"`
- More Latin -> `"en"`
- Empty or ambiguous -> `None`

No LLM or external library used.

## Result on existing data

Both lessons' visual summaries are English (generated by the dense visual analyzer), so after patching:
- `summary_primary`: populated on all 92 evidence refs
- `summary_language`: `"en"` on all refs
- `summary_ru`: `None` on all refs (correctly -- no Russian content was being fabricated)
- `summary_en`: populated on all refs
- `compact_visual_summary`: unchanged
