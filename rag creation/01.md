7. LEXICAL INDEX
==================================================

Create:
pipeline/rag/lexical_index.py

Build a persisted lexical index in Python.
Do not postpone lexical retrieval.

Required behavior
- tokenize title + text + keywords + alias_terms
- keep per-doc unit_type, lesson_id, concept_ids
- support filtering by unit_type, lesson_id, concept_id
- support exact phrase boost
- support concept alias boost
- support multilingual token handling as best practical for v1
- return top-k candidates with score breakdown

Implementation guidance
- use a lightweight BM25-style scorer or equivalent term-based scorer
- persist:
  - vocabulary / df stats
  - doc lengths
  - postings or a compact per-term structure
  - manifest with version info
- keep the scoring abstraction clean so backend can be swapped later

Output manifest
output_rag/index/lexical_index_manifest.json

- evidence-focused retrieval
- concept-focused retrieval

Return structure for each candidate

- unit_type
- raw_lexical_score
- raw_vector_score
- graph_boost
- rerank_score
- final_score
- score_breakdown
- reasons
- resolved_doc
