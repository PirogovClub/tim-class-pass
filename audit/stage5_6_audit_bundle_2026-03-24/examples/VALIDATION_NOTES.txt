Default validation (no flags) checks:
- Manifest + listed files + line counts vs manifest
- JSON parse + required keys + gold/silver tier labels
- No same (target_type, target_id) in both gold and silver exports
- Unique (target_type, target_id) within each gold_*.jsonl / silver_*.jsonl tier file
- gold_rules canonical_family_id must appear in gold_canonical_families.jsonl
- gold_canonical_families member_rule_ids must each appear in gold_rules.jsonl
- If is_duplicate and duplicate_of_rule_id set: without --db, duplicate target must appear in gold_rules.jsonl; with --db, rows must exist in adjudication SQLite
- eval_* rows must reference only target ids present in the corresponding gold exports

Optional CLI flags:
  validate --export-dir DIR --db path/to/adjudication.sqlite
  validate --export-dir DIR --strict-provenance
