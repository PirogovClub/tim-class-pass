## 3. Metric definitions

Authoritative text: **`pipeline/adjudication/metrics_docs.md`** (also copied under `source/pipeline/adjudication/` in this bundle).

Summary: coverage uses inventory targets bucketed by explorer `get_all_docs()` metadata; numerator = materialized tier exists and `tier != unresolved`; proposal acceptance exposes both closed-terminal and all-proposals rates; throughput = decision rows in rolling `7d`/`30d` UTC window; queue backlog = current unresolved queue size and open proposal counts by queue name.

## 7. Known limitations

- Throughput filters decisions by scanning `review_decisions` in Python (correct for ISO timestamps; not a SQL-side window for all dialects).
- Fresh proposal rows inserted via `upsert_proposal` normalize to `NEW`; terminal statuses appear after lifecycle updates (as in production).
- Coverage/flag grouping requires explorer corpus docs; otherwise `explorer_available: false` and empty buckets / empty group tables.
