Role
You are a principal Python engineer modifying an existing trading-video knowledge pipeline.

Mission
Change the framework so that:
1. transcript / spoken theory is the PRIMARY source of truth
2. video is SUPPORTING evidence, not the main source
3. absence of visual evidence does NOT invalidate a clearly stated theory rule
4. visual evidence is still preserved for examples, screenshots, timestamps, and future ML labeling
5. strict validation only requires visual evidence when the rule type actually needs it

Do NOT rewrite the pipeline.
Do NOT remove visual evidence support.
Do NOT weaken provenance.
Do NOT make markdown or visuals the source of truth.

Ground truth policy
- Transcript-first
- Evidence-aware
- Not video-dependent
- Visuals may increase confidence
- Visuals may be required only for explicitly visual/example-driven rules
- Theory rules may be valid without visual backing if transcript support is strong

Existing architecture that must be preserved
- visuals are stored as evidence, not verbose narrative
- markdown is derived from rule_cards + evidence_index, not raw dense narration
- confidence scoring already includes transcript clarity and evidence support as separate signals
- concept resolution already prefers LLM/transcript before visual hints
Preserve this architecture and make it more explicit in code and validation policy.

- if transcript_support_score >= 0.60 and visual_support_score < 0.35 -> transcript_primary

- if transcript_support_score < 0.45 and visual_support_score >= 0.60 and teaching_mode in {"example", "mixed"} -> visual_primary

4) classify_transcript_support_level(score: float) -> str
- >= 0.75 -> strong
- >= 0.45 -> moderate
- else weak

5) classify_visual_support_level(score: float, example_role: str | None) -> str
Suggested:
- no evidence -> none
- illustration -> illustration
- ambiguous_example -> ambiguous
- counterexample -> counterexample
- score >= 0.75 -> strong_example
- else supporting_example

6) should_require_visual_evidence(rule_or_event) -> bool
Return true only when:
- teaching_mode == "example"
or
- evidence_requirement == "required"

Recommended components:

- explicit event type from extraction bucket (definition/rule/condition etc): up to 0.15

- repeated confirmation in nearby text / duplicate support across chunk result: up to 0.15

- candidate visuals present nearby: up to 0.20
- concept match from visuals: up to 0.20

Where concept_anchor_score can be derived from existing concept/subconcept cleanliness.

Important:
- no visual evidence should NOT collapse a clearly transcript-grounded theory event to low confidence
- a strong transcript-grounded event with zero visuals can still be medium/high
- visual support should upgrade confidence, not dominate it

Suggested final event confidence label:
- high: >= 0.72
- medium: >= 0.45
- low: < 0.45

This is the most important behavior change.

Current problem to fix:
rules without visual evidence are treated too broadly as a validation problem.

New export policy:
A rule is export-valid when:
- rule_text exists
- provenance exists
- source_event_ids exist
- confidence exists
- support_basis exists or can be inferred
- if evidence_requirement == "required", then evidence_refs must be non-empty
- if evidence_requirement in {"optional", "none"}, empty evidence_refs are acceptable

Add or modify validation helpers:

1) validate_rule_card()
- do not emit generic "no_evidence" warnings for every rule without evidence
- only warn/error based on evidence_requirement

2) validate_rule_card_for_export()
Implement:
- if evidence_requirement == "required" and evidence_refs empty -> hard error
- if evidence_requirement == "optional" and evidence_refs empty -> no hard error; maybe soft warning
- if evidence_requirement == "none" and evidence_refs empty -> fully valid

3) validate_rule_card_collection_for_export()
- preserve valid theory rules without visuals
- reject only those missing required evidence or broken provenance

4) validate_cross_artifact_references()
- if rule has evidence_refs, they must resolve
- if rule has no evidence_refs and requirement is optional/none, that is valid

5) pipeline/corpus/validator.py validate_lesson()
Replace generic strict no-evidence policy with:
- no warning for theory rules with optional/none requirement
- warning or error only for required-evidence rules lacking refs
- include counts:
  - rules_without_required_evidence
  - theory_rules_without_visuals
  - optional_rules_without_visuals

This is the core semantic fix.
