TASK 3 — BUILD STABLE GLOBAL ID LOGIC
--------------------------------------------------

Create a deterministic global ID utility, for example:

pipeline/corpus/id_utils.py

This utility must generate stable IDs for:
- lesson_id
- event_id
- rule_id
- evidence_id
- node_id
- relation_id

Rules:
1. IDs must be deterministic across reruns
2. IDs must not depend on iteration order
3. IDs must remain stable if unrelated lessons are added later
4. Prefer compositional IDs over random UUIDs

Recommended strategy:
- keep existing lesson_id if already stable
- convert local IDs into global IDs using lesson_id prefixing
- if needed, derive IDs from stable content keys

Examples:
- lesson:<lesson_slug>
- event:<lesson_slug>:<local_event_id>
- rule:<lesson_slug>:<local_rule_id>
- evidence:<lesson_slug>:<local_evidence_id>
- node:<normalized_concept_name>
- relation:<src_node_id>:<relation_type>:<dst_node_id>

Also create normalization helpers:
- slugify lesson key
- normalize concept names
- normalize alias text
- normalize whitespace / punctuation where needed

TASK 5 — BUILD THE CORPUS VALIDATOR
--------------------------------------------------

Create a validator module, for example:

pipeline/corpus/validator.py

Also create a CLI entry point, for example:
- scripts/validate_corpus_inputs.py
or
- python -m pipeline.corpus.validate

Validator responsibilities:

A. Per-lesson validation
- required files exist
- JSON parses successfully
- schema fields match v1 contract
- IDs are present
- lesson IDs are consistent across artifacts
- timestamps are sane if present
- evidence refs referenced by rules exist
- source_event_ids referenced by rules exist
- concept graph structure is valid

B. Cross-lesson validation

C. Warning-level checks

- missing optional fields
- unusually large counts
- rules with no evidence refs
- evidence with no linked rules
- concepts with suspicious alias duplication

The validator must support:
- strict mode: fail on warnings
- normal mode: fail only on hard errors

Output:
- console summary
- machine-readable validation report JSON

Suggested output:
output_corpus/validation_report.json

3. Merge all rule cards into corpus_rule_cards.jsonl

- preserve frame_ids

- preserve timestamps
- preserve linked_rule_ids
- preserve compact visual summary

5. Merge concept graphs into corpus_concept_graph.json
Represent corpus concept graph as:
{
  "nodes": [...],
  "relations": [...]
}

Where:
- concept nodes are deduplicated across lessons by canonical normalized concept key
- per-lesson provenance is preserved on nodes and relations

No LLM calls in this step unless absolutely unavoidable.
