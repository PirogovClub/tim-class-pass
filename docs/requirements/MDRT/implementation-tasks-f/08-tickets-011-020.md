## MDRT-012B: Normalizer — Daily and Monthly Bar Semantics

### Purpose
Add `1D` and `1M` bar normalization to the Normalizer built in MDRT-012A. Daily bar `ts_utc` = session close; monthly bar `ts_utc` = last-trading-day close. Handle early-close sessions correctly.

### Source Docs
- `04-core-pipeline.md` §4.2 (Normalizer — daily/monthly subsection)
- `02-data-models.md` §Daily Bar Semantics, §Monthly Bar Semantics, §session_close_ts_utc rules
- `10-phases.md` §Calendar (early close = 13:00 ET)
- `09-testing.md` §test_normalizer.py (daily/monthly subset)

### Exact Requirements from Docs
- Daily (`1D`): `ts_utc` = session CLOSE time in UTC (not open)
  - Normal close: 16:00 ET → `21:00:00Z` (winter) / `20:00:00Z` (summer)
  - Early close: 13:00 ET → `18:00:00Z` (winter) / `17:00:00Z` (summer)
- Daily: `session_close_ts_utc` = `ts_utc` (non-null, mandatory)
- Daily: `session_date` = the exchange-local trading date
- Monthly (`1M`): `ts_utc` = last trading day's session CLOSE in UTC
- Monthly: `session_date` = last trading day of the month
- Monthly: `session_close_ts_utc` = `ts_utc` (non-null, mandatory)
- Requires `TradingCalendar` to determine exact close time per date

## MDRT-015B: ArchiveWriter — Overlap Detection + Unified Merge

### Purpose
Add the Unified Merge algorithm to the ArchiveWriter: detect overlapping files, prune stale bars, merge, assert no duplicates, and write exactly once.

### Source Docs
- `11-overlap-policy.md` §11.3 (Replace Policy), §11.5 (Unified Merge Step-by-Step), §11.6 (Post-Merge Assert), §11.7 (Catalog Lineage)
- `04-core-pipeline.md` §4.4 (ArchiveWriter — overlap subsection)

### Exact Requirements from Docs
- Before write: query `CatalogManager.find_overlapping_files()`
- If overlaps found, apply Unified Merge: load→prune→merge→write-once
- Old Parquet files deleted BEFORE new file written (never both on disk simultaneously)
- Old `ArchiveFileRecord.superseded_by` = new file ID
- Combined table: sort by `ts_utc` ASC, assert ZERO duplicates
- If duplicates found after merge → `ArchiveWriteError` (before disk write)
- `archive_coverage` updated with merged `first_ts`/`last_ts`/`row_count`
- `data_quality_events` gets `OVERLAP_RESOLVED` INFO event per replaced file
- All steps in `try/except`: if any step fails → `ArchiveWriteError`
