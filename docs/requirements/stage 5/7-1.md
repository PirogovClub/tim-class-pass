REQUIRED FEATURE FAMILIES
================================

The project guidance already pointed to the feature families to start with.
You must implement a practical subset of these, not just list them.

Required feature families to include in the first pass:

1. LEVEL RELATIVE POSITION
- signed distance from reference level
- normalized distance from level
- close relative to level
- high/low penetration relative to level

2. VOLATILITY / SCALE
- ATR or ATR-like volatility normalization
- local range statistics
- volatility regime flag or bucket

3. CANDLE STRUCTURE
- candle body size
- upper wick ratio
- lower wick ratio
- close location within bar
- breakout bar size
- rejection bar shape signals

4. INTERACTION / PERSISTENCE
- number of closes above/below level over allowed local windows
- persistence ratio above/below level
- re-entry/failure back through level
- retest distance and timing

5. PRE-BREAK / PRE-INTERACTION CONTEXT
- consolidation width before anchor
- compression / range narrowing
- pre-anchor trend slope
- pre-anchor momentum proxy

6. VOLUME CONTEXT
- volume vs rolling baseline
- breakout volume percentile or z-score
- abnormal-volume flags if data exists

7. TIME / SESSION CONTEXT
- session bucket
- time-of-day bucket
- day-of-week if present in available data

8. HIGHER-TIMEFRAME ALIGNMENT
- optional HTF directional alignment
- optional HTF position relative to level
- do not make HTF mandatory if absent in current fixtures

These feature families should be tied back to rule concepts where practical.

Examples:
- acceptance -> close persistence above/below level
- false_breakout -> breakout then reversal through level
- volume_confirmation -> breakout volume percentile
- retest -> timing and depth of revisit
- rejection -> wick/body structure + failure to hold level
