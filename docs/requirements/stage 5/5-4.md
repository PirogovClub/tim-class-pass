# Stage 5.4 — Gold / Silver / Bronze Resolution

## Agent Context Document

## Purpose of Stage 5.4

Stage 5.4 builds the **quality-resolution layer** on top of the accepted adjudication foundation and workstation.

By the end of Stage 5.4, the system should be able to:

* assign every supported reviewable item to:

  * `gold`
  * `silver`
  * `bronze`
  * `unresolved`
* explain **why** the item received that tier
* explain **what blocks promotion**
* expose tier state through APIs
* support filtering and counting by tier
* surface tier and blocker information in the Stage 5.3 workstation

Stage 5.4 is where the system becomes a **quality-stratified corpus**, not just a set of review logs and review screens.

---

# Why Stage 5.4 exists

You now have:

* **5.1** adjudication storage and canonical family structure
* **5.2** adjudication APIs and queue APIs
* **5.3** a real review workstation UI

That means the system can already:

* store decisions
* expose decisions
* let humans review items

But it still cannot answer the most important downstream question:

## Which items are trustworthy enough to use, and why?

That is the job of Stage 5.4.

Without Stage 5.4:

* the corpus has no stable quality tiers
* downstream export cannot distinguish trusted vs provisional items
* the future proposal layer has no clear target state
* later ML/eval datasets cannot be assembled safely

So Stage 5.4 is the bridge from **review activity** to **curated corpus quality**.

---

# Stage 5.4 Goals

## Primary goals

* define strict tiering policy
* resolve each supported item to a tier deterministically
* expose explicit blocker reasons
* store or materialize tier state
* expose tier state via stable APIs
* surface tier information in the workstation UI
* prepare the corpus for proposal-assisted review and later export

## Non-goals

Stage 5.4 should **not** become:

* proposal generation
* export pipeline
* model training
* analytics dashboard
* reviewer assignment workflow
* batch re-labeling platform
* policy engine for every future entity type
* large backend refactor

Keep Stage 5.4 focused on **tier resolution**.

# Global Design Principles

## 1) Gold must stay strict

Gold is the high-trust subset.
Do not water it down for volume.

## 2) Unresolved must be operational, not vague

“Unresolved” should mean the system can tell you exactly what is still missing or blocking promotion.

## 3) Tiering must be deterministic

Given the same adjudicated state, the same item must resolve to the same tier every time.

## 4) Tiering must be explainable

Every tier result should carry:

* tier
* reasons
* blockers
* supporting status fields

## 5) Linked structure matters

Rule quality is not always independent.
Canonical family validity and linked reviewed state may affect promotion.

## 6) Bronze is not failure

Bronze is still useful for:

* discovery
* search
* candidate generation
* future review

Bronze simply means “not trusted enough yet.”

## 7) Tiering should build on adjudication, not replace it

Stage 5.4 consumes the output of 5.1–5.3.
It does not invent a parallel truth system.

# Tier Definitions

## Gold

High-confidence, strongly adjudicated, safe for trusted downstream use.

Typical characteristics:

* meaningful adjudication exists
* no unresolved blockers
* no ambiguity/defer state
* no unsupported state
* canonical linkage valid if relevant
* linked reviewed state not blocking

## Silver

Useful and structured, but not fully trusted.

Typical characteristics:

* partially reviewed or structurally clean
* no hard blockers
* not strong enough for Gold yet
* still useful for weaker downstream use and prioritization

## Bronze

Raw or weakly reviewed.

Typical characteristics:

* exists in the corpus
* little or no strong adjudication
* not clearly invalid, but not trusted
* still useful for discovery/search/candidate generation

## Unresolved

Needs more review before stable quality assignment.

Typical characteristics:

* `needs_review`
* `ambiguous`
* `defer`
* contradictory linked state
* blocking canonical/family problems
* structurally incomplete or invalid review context

# Recommended Tier Policy

You may adjust exact rules, but document them explicitly.

## Rule-card resolution policy

### Gold rule card

A `rule_card` may become Gold only if all of these are true:

* item exists in authoritative corpus
* adjudication state exists
* current state is not:

  * `needs_review`
  * `ambiguous`
  * `defer`
  * `unsupported`
* if marked duplicate, its duplication state is valid and not broken
* if family-linked, the family exists and linkage is valid
* blocking linked states do not invalidate it
* no tier blocker remains

### Silver rule card

A `rule_card` may become Silver if:

* item exists in authoritative corpus
* it is not Gold
* it has no hard invalidating blockers
* it has enough structure/adjudication to be above Bronze
* it is not currently unresolved

### Bronze rule card

A `rule_card` may become Bronze if:

* item exists in authoritative corpus
* it lacks sufficient adjudication for Silver/Gold
* it is not clearly invalid
* it is not blocked into unresolved by active adjudication states

### Unresolved rule card

A `rule_card` must be Unresolved if:

* no adjudication exists and policy says first review is required
* state is `needs_review`
* state is `ambiguous`
* state is `defer`
* canonical/family linkage is broken
* linked blockers make quality judgment incomplete

## Evidence-link policy

### Gold evidence link

* explicitly reviewed
* strong support
* no invalidating blocker

### Silver evidence link

* reviewed and partially acceptable
* not strong enough for Gold
* no hard blocker

### Bronze evidence link

* exists but weakly reviewed or minimally trusted

### Unresolved evidence link

* needs review
* conflicting or incomplete support state
* structurally invalid context

## Concept-link policy

### Gold concept link

* explicitly reviewed and valid

### Silver concept link

* acceptable but not strongly adjudicated

### Bronze concept link

* present but weakly reviewed

### Unresolved concept link

* invalid or unresolved state preventing confident classification

## Related-rule relation policy

### Gold related relation

* explicitly reviewed and valid

### Silver related relation

* acceptable but not strongly adjudicated

### Bronze related relation

* present but weakly reviewed

### Unresolved related relation

* invalid or unresolved state preventing confident classification
