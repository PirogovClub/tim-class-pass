REQUIRED CLASS SEMANTICS
================================

Use these class semantics as the starting direction.

You must refine them and make them explicit.

acceptance_above:
- price meaningfully establishes above a reference level
- persistence above level matters
- immediate failure back below should invalidate

acceptance_below:
- mirror of acceptance_above for below-level behavior

false_breakout_up:
- price breaks above level but fails and returns below within the allowed horizon

false_breakout_down:
- mirror of false_breakout_up

rejection:
- price tests a level and rejects away without satisfying acceptance behavior

no_setup:
- the candidate window does not satisfy any defined setup class
- use when the task is applicable but no target setup is present

ambiguous:
- multiple classes appear plausible
- evidence is insufficient
- the anchor/level is unclear
- market structure is too noisy for deterministic assignment
- use only when classification should be withheld, not when there is simply no setup

You must make the class boundaries cleaner than the bullet notes above.
