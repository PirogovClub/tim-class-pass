## 9. Strong recommendation on metric definitions

Before coding the endpoints, define the metric meanings clearly.

At minimum document these definitions:

### 9.1 “Coverage by lesson”

Recommended definition:

* numerator: number of supported review targets in the lesson that are **not unresolved**
* denominator: total supported review targets in the lesson

Also include raw counts, not just a percentage.

### 9.2 “Coverage by concept”

Recommended definition:

* numerator: number of supported review targets under that concept that are **not unresolved**
* denominator: total supported review targets under that concept

### 9.3 “Proposal acceptance rate”

Recommended definition:

* accepted proposals / proposals that reached a terminal disposition
  or
* accepted proposals / all proposals generated

Pick one and document it clearly.
If possible, expose both:

* `acceptance_rate_closed`
* `acceptance_rate_all`

### 9.4 “Review throughput”

Recommended definition:

* number of adjudication decisions created during a defined window, for example 7d or 30d

### 9.5 “Queue backlog”

Recommended definition:

* current number of open items in the relevant queue at computation time

These definitions are important because the success criteria explicitly require stable documented metric definitions.
