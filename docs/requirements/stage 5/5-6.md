**This task is NOT done when coding is finished.**

## 7. Export schema requirements

You must define explicit export schemas.

### 7.1 General rules

* Export formats must be stable and documented.
* Use JSONL for row-like artifacts unless the codebase already has a stronger accepted export format.
* Each row must contain:

  * stable ID
  * target type
  * tier
  * provenance
  * canonical/adjudication linkage if relevant
  * schema version
  * export version metadata where useful

### 7.2 Gold export rules

Gold exports should include only items that satisfy the strict reviewed/tier policy already established by prior stages.

Do **not** include:

* unresolved items
* unsupported items
* clearly invalid items
* blocked items
* rejected items unless the export explicitly documents a rejected dataset, which is not part of this step

### 7.3 Silver export rules

Silver exports must be separate and explicitly labeled Silver.
Do not quietly combine Gold and Silver in one “trusted” file.

### 7.4 Provenance fields

Each exported row should preserve enough provenance for downstream trust:

* `lesson_id`
* source target ID
* linked evidence IDs if relevant
* linked canonical family ID if relevant
* review/tier policy version
* export timestamp or manifest linkage

The project’s cross-step rule is that provenance is never optional.

## 8. Inclusion and exclusion policy

Before writing export code, define export policy clearly in code and docs.

### 8.1 Gold inclusion

Gold export includes only:

* supported reviewed target types
* `tier == gold`
* structurally valid items
* items with required provenance present
* items not blocked by export validation rules

### 8.2 Silver inclusion

Silver export includes only:

* supported target types
* `tier == silver`
* structurally valid items
* clearly marked as Silver
* not confused with Gold

### 8.3 Exclusions

At minimum exclude:

* unresolved
* unsupported
* invalid
* missing required provenance
* broken references
* broken canonical linkage where required
* contradictory data that fails validation

Write this policy down in `export_policy.py` and in export docs.
