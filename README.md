# Multimodal YouTube Video Transcript Enrichment Pipeline

Enriches a YouTube video's spoken transcript with rich **visual descriptions** generated by analyzing every second of video footage. The result is a self-contained enriched `.vtt` file and a standalone `video_commentary.md` screenplay ‚Äî both usable as LLM context, search indexes, or study aids.

---

## Requirements

| Tool | Purpose |
|------|---------|
| **Python 3.11+** via `uv` | Run the pipeline |
| **FFmpeg** (system PATH) | Extract frames from video |
| **yt-dlp** | Download video + subtitles |

### Environment Setup

```bash
cp .env.template .env
```

Edit `.env`:
```env
LLM_PROVIDER=antigravity   # antigravity | openai | gemini
OPENAI_API_KEY=sk-...      # if using openai
GOOGLE_API_KEY=...         # if using gemini
```

> **Antigravity mode** means the AI agent (you are reading this) performs all visual analysis directly using its `view_file` tool ‚Äî no external API calls needed.

## How the Pipeline Works

The pipeline has 3 sequential steps controlled by `main.py`.

```
YouTube URL / existing video_id
        ‚îÇ
        ‚ñº
[Step 0] downloader.py         ‚Äî Download .mp4 + .vtt via yt-dlp  (skipped if --video_id)
        ‚îÇ
        ‚ñº
[Step 1] dense_capturer.py     ‚Äî Extract 1 frame/second ‚Üí frames_dense/frame_NNNNNN.jpg
        ‚îÇ
        ‚ñº
[Step 2] dense_analyzer.py     ‚Äî Agent analyzes frames in batches of N
        ‚îÇ                         Writes per-frame .txt files + dense_analysis.json
        ‚îÇ                         (exits with code 10 between batches for agent to respond)
        ‚ñº
[Step 3] deduplicator.py       ‚Äî Groups frames into scenes, asks agent to write
                                  polished descriptions ‚Üí *_enriched.vtt + video_commentary.md
```

### Step 2 Detail ‚Äî Batch Analysis

The pipeline writes a prompt file and exits (code 10) after each batch:

```
dense_batch_prompt_000001-000010.txt  ‚Üê agent reads this
dense_batch_response_000001-000010.json  ‚Üê agent writes this
```

The prompt lists 10 frame paths. The agent opens each `.jpg`, writes a full description + delta, saves the JSON, then re-runs the pipeline. The pipeline merges the response and continues to the next batch.

Each `frames_dense/frame_NNNNNN.txt` looks like:
```
[Description]
MetaTrader 4 EUR/USD Daily chart. Downtrend from 1.2370 to 1.1580.
Red horizontal levels at 1.1862, 1.1960, 1.2270.

[Delta]
Added: New red horizontal line at 1.1960
```

### Step 3 Detail ‚Äî Deduplication

After all frames are analyzed, the deduplicator groups consecutive identical frames into scenes (56 scenes for a 666-frame video). It writes `dedup_prompt.txt` and exits.

The agent writes `dedup_response.json` with one polished paragraph per scene, then re-runs once more to produce the final outputs.

## Running the Pipeline

### From a new YouTube URL

```bash
uv run main.py --url "https://www.youtube.com/watch?v=VIDEO_ID" --batch-size 10
```

### From an existing video folder (video + VTT already present)

```bash
uv run main.py --video_id VIDEO_ID --batch-size 10
```

### Force re-extract frames

```bash
uv run main.py --video_id VIDEO_ID --recapture --batch-size 10
```

### CLI Reference

| Flag | Default | Description |
|------|---------|-------------|
| `--url URL` | ‚Äî | YouTube URL to download and process |
| `--video_id ID` | ‚Äî | Use existing `data/<ID>/` folder (skip download) |
| `--provider` | `antigravity` | LLM provider: `antigravity`, `openai`, `gemini` |
| `--batch-size N` | `10` | Frames per agent batch (5‚Äì20 recommended) |
| `--recapture` | off | Force re-extract frames even if already done |

## Starting the Pipeline as an Agent

When you (an AI agent) are asked to run this pipeline on a video, follow these steps:

### 1. Trigger the workflow

```
/multimodal_transcript_pipeline
```

This workflow has `// turbo-all` ‚Äî all shell commands auto-run without approval.

### 2. Run the initial pipeline command

```bash
uv run main.py --video_id <VIDEO_ID> --batch-size 10
```

- If downloading: use `--url` instead.
- The pipeline will exit immediately after writing the first batch prompt.

### 3. Loop until Step 2 is complete

Read the `dense_batch_prompt_NNN-MMM.txt` file. It tells you which frames to analyze.

For each frame in the batch:
1. `view_file` on the absolute path to the `.jpg`
2. Write description (what is on screen) + delta (what changed vs. previous)
3. Save `dense_batch_response_NNN-MMM.json`
4. Re-run the pipeline command

**Do this autonomously in a loop** ‚Äî do not wait for user confirmation between batches. Survey keyframes ahead of time to understand scene structure, then write entire segments efficiently.

```json
{
  "000001": { "description": "...", "delta": "Scene start" },
  "000002": { "description": "...", "delta": "No change" }
}
```

### 4. Handle the deduplication step

After all 666+ frames are processed, the pipeline writes `dedup_prompt.txt`.

1. Read `dedup_prompt.txt` ‚Äî it lists 50‚Äì80 scenes with all their deltas.
2. Write `dedup_response.json` ‚Äî one polished paragraph per scene timestamp.
3. Re-run the pipeline one final time.

### 5. Verify completion

```bash
# Check frame count
(Get-ChildItem "data\<VIDEO_ID>\frames_dense" -Filter "*.txt").Count

# Check outputs exist
Get-ChildItem "data\<VIDEO_ID>" -Filter "*_enriched.vtt"
Get-ChildItem "data\<VIDEO_ID>" -Filter "video_commentary.md"
```

Read the first 30 lines of `video_commentary.md` to confirm the narrative makes sense.

## Tips for Efficient Agent Analysis

| Situation | Strategy |
|-----------|----------|
| Static lecture scene (slides, chart) | Survey every 20‚Äì50 frames, write "No change" for identical ones |
| Active drawing / annotations | View every frame ‚Äî changes happen every 1‚Äì3 seconds |
| Transition detected | Mark as `"Scene change: <description>"` in delta |
| Long repeated scene | Write one batch response covering all frames with "No change" |

**Batch size guidance:** Default 10 is safe. Increase to 20 for fast-moving content you've already surveyed. Reduce to 5 for dense annotation scenes.
